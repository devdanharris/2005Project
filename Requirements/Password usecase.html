Title: User Password Authentication
Primary Actor: User
Stakeholders and Interests: User - trying to gain access into SuD.
Precondition: User already registered for SuD
Trigger: User clicks login button.
Main success scenario:

  1. User clicks login button after filling out credentials.
  2. SuD verifies that user is a registered user.
  3. User gains access to all channels they are a part of.

Extensions:

  1. 2a. User put in credentials in the wrong password:
    1. 2a1. SuD notifies user of wrong credentials.
    2. 2a2. User re-enters credentials correctly
    3. 2a3. User initiates User Login use case
  2. 2a. User is not a formally registered student or a special guest:
    1. 2a1. SuD notifies user of wrong credentials.
    2. 2a2. User initiates User Register use case
